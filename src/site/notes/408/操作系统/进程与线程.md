# 进程
- 定义：
	- 程序的一次执行过程
	- 程序及其数据在处理机上顺序执行时所发生的活动
	- 具有独立功能的程序在一个数据集合上运行的过程，系统进行资源分配和调度的一个独立单位（如果没有线程，是最基本的单位）
- 特点
	- 动态性：生命周期
	- 并发性：多个进程共存于内存
	- 独立性：独立运行，分配资源，调度基本单位
	- 异步性：进程间相互制约
- 进程实体/映像（静态）：
	- 由程序段，相关数据段和PCB组成
	- <mark style="background: #FF5582A6;">PCB是进程存在的唯一标志</mark>

- 运行态：单处理机中每个时刻只有一个进程在运行态
- 就绪态：获取了处理机之外的所有所需资源，形成就绪队列
- 阻塞态/等待态：等待某一事件（如资源），形成阻塞队列
- 创建态：进程正申请空白PCB，填入用于控制管理进程的信息，等待分配资源
- 结束态：进程正在正常结束或因其他原因退出，先被置为结束态后释放资源回收

```ad-caution
就绪态与阻塞态等待的目的不同，运行态频繁切换到就绪态，较少切换到阻塞态
```

- 就绪态$\Rightarrow$运行态：获得处理机资源
- 运行态$\Rightarrow$就绪态：时间片用完或被剥夺
- 运行态$\Rightarrow$阻塞态：进程<mark style="background: #FF5582A6;">主动</mark>请求资源或等待事件，进程通过系统调用请求操作系统服务
- 阻塞态$\Rightarrow$就绪态：时间到来，进程<mark style="background: #ABF7F7A6;">被</mark>唤醒

## 进程的组织
- PCB
	- 组成：
		- 进程描述信息
		- 进程控制和管理信息
		- 资源分配清单
		- 处理机相关信息/处理机上下文
	- 组织方式：链接，索引
- 程序段
	- 可在不同进程间共享
- 数据段
	- 原始、经加工数据

## 进程控制

### 创建
- 父进程创建子进程
- 子进程可继承父进程所有资源，被撤销时归还
- 父进程被撤销时通常也被撤销
- 创建原语：
	- 分配唯一的进程标示号，尝试申请空白PCB
		- PCB是有限的
		- 申请失败则创建失败
	- 分配所需资源
		- 来源为操作系统或父进程
		- 资源不足维持创建态
	- 初始化PCB
		- 初始化标志信息，处理机状态信息，处理机控制信息，进程优先级
	- 能接纳的就绪队列将其插入

### 终止
- 引起终止的事件
	- 任务完成，正常结束
	- 发生异常，异常结束
	- 应外界（操作员，操作系统，父进程）请求，外界干预终止
- 终止原语：
	- 根据标识符检索PCB，读出状态
	- 若执行，<mark style="background: #FF5582A6;">立刻</mark>终止执行，重分配处理机资源
	- 终止子孙进程（如果有）
	- 归还所有资源给父进程或操作系统
	- 将PCB从所在队列/链表删除

### 阻塞
- 进程主动调用阻塞原语
- 阻塞原语：
	- 根据标示号定位PCB，读取状态
	- 为运行态，保护现场，转为阻塞态，停止运行
	- 将PCB插入对应事件等待序列，冲分配处理机资源

### 唤醒
- 进程被其他相关进程调用唤醒原语唤醒
- 唤醒原语：
	- 在事件等待队列中找到相应PCB
	- 移出等待队列，置为就绪态
	- 插入就绪队列等待调度

```ad-caution
阻塞和唤醒原语一一对应，否则永远阻塞
```

## 进程的通信
- 低级通信：PV操作
- 高级通信：
	- 共享存储
		- 需要同步互斥工具（如PV）
		- 分为：
			- 低级方式：数据结构
			- 高级方式：存储区
		- 操作系统只提供存储空间与同步互斥工具
		- 进程空间一般独立，需要特殊系统调用
		- 同一进程内线程自然共享
		- 适用于同一台主机的不同进程
	- 消息传递
		- 格式化的信息为单位

# 线程
- 系统独立调度和分派的基本单位
- 有唯一标识符和线程控制块TCB，记录寄存器和栈现场
- 几乎不拥有系统资源，共享进程资源（地址空间，全局变量）
- 并发性
- 不同线程可执行相同程序

```ad-caution
进程为CPU外系统资源的分配单元
线程为处理机的分配单元
```

- 与进程的区别

	|                  | 进程               | 线程                             |
	| ---------------- | ------------------ | -------------------------------- |
	| 调度             |                    | 独立调度基本单位，切换代价小     |
	| 并发             | 可                 | 可，进一步增加系统并发性         |
	| 拥有资源         | 拥有资源的基本单位 | 几乎不拥有系统资源，访问进程资源 |
	| 独立性           | 独立地址空间与资源 | 共享进程地址空间与资源           |
	| 系统开销         | 分配回收切换开销大 | 小，统一进程内时同步通信也容易   |
	| 支持多处理机系统 | 一个进程一个处理机 | 多个线程分布于不同处理机一个进程 | 
- 状态转换：执行，就绪，阻塞
- 组织与控制
	- 线程组织块TCB
		- 线程标识符，一组寄存器，线程运行状态，优先级，专有存储区，堆栈指针
		- 线程间可以修改堆栈（进程资源空间共享）
	- 创建
		- 用户程序启动时有初始化线程执行
	- 终止
		- 一些线程（主要为系统进程）建立后不会终止
		- 线程终止布里克释放资源，同进程中其他线程执行分离函数才分离，腾出资源
		- 被终止但尚未释放资源的线程仍可被其他线程调用并恢复运行

		- 每个作业只调入调出一次
		- 属于多到批处理系统
	- 中级调度（内存调度）
		- 将暂时不能运行进程调入外存挂起
		- 具备运行条件后重新调入就绪队列
		- 存储器管理的对换功能
		- 提高内存利用率
	- 低级调度（进程调度）
		- 从就绪队列中选一个进程执行，分配处理机
- 调度目标
	- CPU利用率：$$\text{CPU利用率}=\frac{\text{CPU有效工作时间}}{\text{CPU有效工作时间}+\text{CPU空闲等待时间}}$$
	- 系统吞吐量：单位时间内CPU完成作业的数量
	- 周转时间：作业提交到完成的时间（作业等待+就绪排队+处理机运行+输入/输出时间）
		- $$\text{周转时间}=\text{作业完成时间}-\text{作业提交时间}$$
		- 带权周转时间：$$\text{带权周转时间}=\frac{\text{作业周转时间}}{\text{作业实际运行时间}}$$
	- 等待事件：进程处于等待处理机时间之和
	- 响应时间：用户提交请求到系统首次响应的时间
- 调度程序
	- 排队起
	- 分排期
	- 上下文切换器（两对操作）
- 调度时机
	- 不能调度
		- 处理中断过程中
		- 进程进入操作系统内核临界区时
		- 其他需要完全屏蔽中断的原子操作过程中
	- 能调度
		- （非剥夺）发生引起调度条件，且当前进程无法继续执行
		- （剥夺）中断或自陷处理结束后，返回被中断进程用户态程序执行现场前
- 进程调度方式
	- 非抢占/剥夺式调度
	- 抢占/剥夺式调度
- 闲逛进程
	- 切换时无就绪进程
	- 不会被阻塞
	- 不断测试中断
	- 优先级最低，有新就绪进程立刻让出
- 线程调度
	- 用户级线程调度：对内核透明，快
	- 内核级线程调度：内核赋予时间片，通常无视所属进程，慢

## 典型调度算法
- FCFS 先来先服务
	- 作业/进程调度
	- 非抢占
	- 简单
	- 效率低
	- 对长作业有利，对短作业不利
	- 有利于CPU繁忙型作业（长作业），不利于I/O繁忙型作业（短作业）
- SJF 短作业优先
	- 作业调度
	- 非抢占
	- 平均等待时间，平均周转时间最少
	- 对长作业不利，"饥饿"
	- 不考虑紧迫性
	- "短"为估计，实际情况不定
- 优先级调度算法
	- 作业/进程调度
	- 分类：
		- 按是否抢占：
			- 非抢占式优先级调度算法
			- 抢占式优先级调度算法
		- 按优先级变化：
			- 静态优先级
			- 动态优先级
	- 优先级设置：
		- 系统进程>用户进程
		- 交互性/前台进程>非交互性/后台进程
		- I/O进程>计算型进程
- 高响应比优先算法
	- 综合 FCFS SJF
	- 每次调度计算响应比
	- 响应比$R_{p}=\frac{\text{等待事件}+\text{要求服务时间}}{\text{要求服务时间}}$
- 时间片轮转调度算法
	- 适用于分时系统
	- 所有就绪进程按FCFS
	- 时间片长短因素：系统响应时间，就绪队列中进程数目，系统处理能力
- 多级队列调度算法
	- 多个就绪队列，多种调度方法
	- 多处理机中，可以一个处理机一个就绪队列
- 多级反馈队列调度算法
	- 综合 时间片轮转调度算法 和 优先级调度算法
	- 实现
		- 多个就绪队列，每个进程从第一个队列进入，出后依次进入下一个
		- 各个队列优先级$\searrow$，时间片大小$\nearrow$
		- 每个队列FCFS，最后一个队列时间片轮转
		- 队列间按队列优先级调度，高优先级空调度第优先级
			- 低优先级处理时高优先级进入，低优先级当前进程置于原队尾，转让处理机
	- 优点：
		- 终端型作业用户：短作业优先
		- 短批处理作业用户：周转时间较短
		- 长批作业处理用户：不会长期无法执行
- 总结
	
	|              | 先来先服务   | 短作业优先                 | 高响应比优先     | 时间片轮转                           | 多级反馈队列 |
	| ------------ | ------------ | -------------------------- | ---------------- | ------------------------------------ | ------------ |
	| 是否可抢占   | $\times$     | $\checkmark$               | $\checkmark$     | $\checkmark$                         | 队内算法不定 |
	| 是否可非抢占 | $\checkmark$ | $\checkmark$               | $\checkmark$     | $\times$                             | 队内算法不定 |
	| 优点         | 公平易实现   | 平均等待时间最少，效率最高 | 兼顾长短作业     | 兼顾长短作业，响应时间较好，可行性强 |              |
	| 缺点         | 不利于短作业 | 长作业饥饿，估计时间不确定 | 计算响应比开销大 | 平均等待时间长，上下文切换浪费时间   | 无           |
	| 适用         | 无           | 作业调度，批处理系统       | 无               | 分时系统                             | 相当通用     |
	| 默认决策模式 | 非抢占       | 非抢占                     | 非抢占           | 抢占                                 | 抢占         |

# 死锁
- 产生原因
	- 系统资源竞争
	- 进程推进顺序非法
- 必要条件
	- 互斥
	- 不剥夺
	- 请求并保持
	- 循环等待

## 死锁处理
|          | 资源分配策略       | 可能模式                                 | 主要优点                                     | 主要缺点                                                 |
| -------- | ------------------ | ---------------------------------------- | -------------------------------------------- | -------------------------------------------------------- |
| 死锁预防 | 保守，宁可资源闲置 | 一次请求所有资源，资源剥夺，资源按需分配 | 适用突发式处理的进程，不必剥夺               | 效率低，进程初始化时间延长；剥夺次数过多；不便申请新资源 |
| 死锁避免 | 折中               | 寻找可能安全允许顺序                     | 不必进行剥夺                                 | 需要知道将来资源需求；进程不能被长时间阻塞               |
| 死锁检测 | 宽松，允许就分配   | 定期检查是否发生死锁                     | 不延长进程初始化时间，允许对死锁进行现场处理 | 剥夺接触死锁，造成损失                                   | 

### 死锁预防
- 破坏互斥条件
	- 不可行，不鼓励
- 破坏不剥夺条件
	- 复杂
	- 用于易恢复的资源
- 破坏请求并保持条件
	- 预先静态分配，进程运行前一次性申请所需资源
	- 资源严重浪费，可能导致饥饿
- 破坏循环等待条件
	- 顺序资源分配法，按递增序申请
	- 限制新设备增加；顺序不同时造成浪费；编程麻烦

### 死锁避免
- 安全状态
	- 是当前进程能逐个获得资源完成任务并归还资源的资源分配顺序

#### 银行家算法
- 数据结构
	- 可利用资源向量 `Available[m]`
	- 最大需求矩阵 `Max[n][m]`
	- 分配矩阵 `Allocation[n][m]`
	- 需求矩阵 `Need[n][m]
		- 满足 `Need[i][j] = Max[i][j] - Allocation[i][j]`
- 银行家算法描述
	- 进程请求资源矩阵 `Request[n][m]`
	```pseudo-code
	1. if Request[i][j]<=Need[i][j]:
			goto 2.
		else:
			ERROR;
	2. if Request[i][j]<=Available[i][j]:
			goto 3.
		else:
			Let process wait...
	3. Try to allocate resources fot process. Then
		Available[i][j] -= Request[i][j]
		Allocation[i][j] += Request[i][j]
		Need[i][j] -= Request[i][j]
	4. Safety Check...
		if SAFE:
			Officially allocate.
		else:
			Undo allocatino, let process wait...
	```
- 安全性算法描述
	- 工作向量`Work[i][j] = Available[i][j]`
	```pseudo-code
	1. Initiate empty safety sequence;
	2. Find row in Need that satisfies:
		a. Process not in safety sequence;
		b. Smaller than Work;
		if FOUND:
			Add it to safety sequence;
		else:
			goto 4.
	3. After process is done and resources gets released:
		Work += Allocation[i]
		goto 2.
	4. if safety has all processes:
			SAFE!
		else:
			UNSAFE!
	```

### 死锁检测与删除
- 资源分配图
- 死锁解除
	- 资源剥夺法：挂起某些死锁进程，抢占资源并分给其他死锁进程
		- 应防止被挂起资源饥饿
	- 撤销进程法：强制撤销部分或所有死锁进程，抢占资源
		- 可按进程优先级或撤销进程代价顺序执行
	- 进程回退法：
		- 让若干个进程回退到足以回避死锁的地步
			- 自愿释放非剥夺
			- 系统保持历史信息并设置还原点
